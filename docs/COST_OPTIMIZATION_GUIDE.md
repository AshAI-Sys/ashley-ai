3. Create API key
4. Copy key
   ‚úÖ **Done!**

### Step 4: Storage (10 mins)

2. Create R2 bucket
3. Get API keys
4. Copy keys
   ‚úÖ **Done!**

### Step 5: Deploy (30 mins)

2. Login: `vercel login`
3. Deploy admin: `cd services/ash-admin && vercel --prod`
4. Deploy portal: `cd ../ash-portal && vercel --prod`
5. Add environment variables in Vercel dashboard
   ‚úÖ **Done!**

### Step 6: Test (10 mins)

1. Visit your site
2. Test login
3. Create order
4. Upload file
   ‚úÖ **All working!**

## üéì TIPS PARA MAS TIPID

### Tip 1: Start Free

- Use ALL free tiers first
- Upgrade lang kung kailangan
- Monitor usage weekly

### Tip 2: Cache Everything

- Use Redis caching
- Reduce database queries
- Save $$$ on database

### Tip 3: Compress Files

- Compress images before upload
- Save 70-90% storage
- Lower storage costs

### Tip 4: Batch Emails

- Send emails in batches
- Don't exceed free limits
- Use transactional only

### Tip 5: Monitor Usage

- Check usage weekly
- Set alerts at 80%
- Upgrade before limits

### Tip 6: Optimize Database

- Archive old data
- Delete unused records
- Keep database small

### Tip 7: Scale Smart

- Don't upgrade everything
- Upgrade only what's needed
- Save money where you can
